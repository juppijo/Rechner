Mittel 01 / 07 · CSS

CSS Flexbox – Das flexible Layout

Flexbox löst das jahrelange Problem der vertikalen & horizontalen Ausrichtung in CSS. Mit einer Zeile display: flex wird ein Container zum Flex-Container, und seine direkten Kinder werden zu Flex-Items.

Flex-Container

Wird durch display: flex aktiviert. Steuert die Anordnung aller direkten Kinder mit Properties wie justify-content, align-items, flex-direction.

Flex-Items

Die direkten Kinder des Containers. Mit flex: 1 wachsen sie gleichmäßig. align-self überschreibt die Container-Ausrichtung für einzelne Items.

layout.css
/* Flex-Container */
.container {
  display: flex;
  flex-direction: row;          /* row | column */
  justify-content: space-between; /* Hauptachse */
  align-items: center;           /* Querachse */
  gap: 16px;                      /* Abstand zwischen Items */
  flex-wrap: wrap;               /* Zeilenumbruch erlauben */
}

/* Flex-Items */
.item {
  flex: 1;           /* flex-grow: 1, flex-shrink: 1 */
  flex: 0 0 200px;  /* feste Breite 200px */
  align-self: flex-end; /* nur dieses Item */
}
INFO: justify-content wirkt auf die Hauptachse (horizontal bei row), align-items auf die Querachse (vertikal bei row). Das zu verwechseln ist der häufigste Flexbox-Fehler.

Mittel 02 / 07 · CSS

CSS Grid – Zweidimensionale Layouts

CSS Grid ist das mächtigste Layout-System in CSS. Im Gegensatz zu Flexbox (1D) arbeitet Grid in zwei Dimensionen gleichzeitig – Zeilen und Spalten.

grid.css
.grid {
  display: grid;

  /* 3 Spalten: 1fr 2fr 1fr */
  grid-template-columns: 1fr 2fr 1fr;

  /* Automatische Spalten (responsive!) */
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));

  grid-template-rows: auto 1fr 60px;
  gap: 20px;
}

/* Item überspannt mehrere Zellen */
.header { grid-column: 1 / -1; } /* volle Breite */
.sidebar { grid-row: 2 / 4; }     /* 2 Reihen */

/* Named Grid Areas */
.layout {
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
}
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
TIPP: Die Funktion repeat(auto-fit, minmax(200px, 1fr)) macht ein Grid automatisch responsiv – ohne Media Queries!

Mittel 03 / 07 · CSS

CSS Animationen & Transitions

transition erzeugt weiche Übergänge bei Zustandsänderungen (z.B. Hover). @keyframes definiert komplexe mehrstufige Animationen, die automatisch ablaufen.

animation.css
/* ── Transitions ── */
.button {
  background: blue;
  transition: background 0.3s ease,
              transform  0.2s cubic-bezier(.4,0,.2,1);
}
.button:hover {
  background: red;
  transform: scale(1.05) translateY(-2px);
}

/* ── Keyframe-Animation ── */
@keyframes pulsieren {
  0%   { transform: scale(1);    opacity: 1; }
  50%  { transform: scale(1.1);  opacity: 0.8; }
  100% { transform: scale(1);    opacity: 1; }
}
.puls {
  animation: pulsieren 2s ease-in-out infinite;
  animation-delay: 0.5s; /* Verzögerung */
}

Fortgeschritten 04 / 07 · JavaScript

DOM – Fortgeschrittene Manipulation

Das Document Object Model (DOM) ist die programmatische Repräsentation deiner HTML-Seite. Fortgeschrittene Entwickler nutzen mehr als nur getElementById – sie arbeiten mit Query-Selektoren, Event-Delegation und dynamischem Element-Erstellen.

dom.js
// ── Moderne Selektion ──
const btn  = document.querySelector('.mein-btn');
const alle = document.querySelectorAll('li.aktiv');

// ── Element erstellen & einfügen ──
const li = document.createElement('li');
li.textContent = 'Neuer Eintrag';
li.classList.add('aktiv', 'neu');
document.querySelector('ul').appendChild(li);

// ── Event-Delegation (effizient!) ──
document.querySelector('.liste')
  .addEventListener('click', (e) => {
    if (e.target.matches('li')) {
      e.target.classList.toggle('erledigt');
    }
  });

// ── data-Attribute ──
btn.dataset.userId = 42;   // → data-user-id="42"
const id = btn.dataset.userId;
PERFORMANCE: Verwende Event-Delegation statt Event-Listener auf jedem Element. Ein Listener auf dem Parent-Element ist effizienter als 100 Listener auf 100 Kinder-Elementen.

Fortgeschritten 05 / 07 · JavaScript

Async / Await & die Fetch API

JavaScript ist asynchron – es wartet nicht auf langsame Operationen (Netzwerk, Dateien). Mit async/await schreibst du asynchronen Code, der wie synchroner Code aussieht und lesbar bleibt.

fetch.js
// ── Fetch-API mit async/await ──
async function ladeDaten() {
  try {
    // await pausiert bis die Response kommt
    const response = await fetch('https://api.example.com/users');

    // HTTP-Fehler prüfen
    if (!response.ok) throw new Error(`HTTP Fehler: ${response.status}`);

    // JSON parsen
    const daten = await response.json();
    console.log(daten);
    return daten;

  } catch (fehler) {
    console.error('Fehler:', fehler);
  }
}

// ── Mehrere Requests parallel ──
const [user, posts] = await Promise.all([
  fetch('/api/user').then(r => r.json()),
  fetch('/api/posts').then(r => r.json()),
]);
INFO: async vor einer Funktion macht sie asynchron und sie gibt immer ein Promise zurück. await kann nur innerhalb einer async-Funktion benutzt werden.

Fortgeschritten 06 / 07 · Browser APIs

localStorage & sessionStorage

Browser-APIs ermöglichen das Speichern von Daten direkt im Browser – kein Server nötig! localStorage ist dauerhaft (bleibt nach Browser-Neustart), sessionStorage gilt nur für die aktuelle Tab-Session.

storage.js
// ── localStorage ──
localStorage.setItem('name', 'Max');
const name = localStorage.getItem('name'); // 'Max'
localStorage.removeItem('name');
localStorage.clear();                       // Alles löschen

// ── Objekte speichern (JSON) ──
const user = { id: 1, name: 'Anna', punkte: 42 };
localStorage.setItem('user', JSON.stringify(user));

// Laden & parsen
const geladen = JSON.parse(
  localStorage.getItem('user') ?? 'null'
);

// ── Verfügbar prüfen ──
function getItem(key, fallback = null) {
  try {
    return JSON.parse(localStorage.getItem(key)) ?? fallback;
  } catch { return fallback; }
}
